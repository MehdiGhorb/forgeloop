### STEP 2: Analyze User Requirements and Create Personas
- Create detailed user personas based on requirements
- Identify user goals, motivations, and pain points
- Map user journeys and key scenarios
- Identify primary, secondary, and tertiary user flows
- Document user stories and acceptance criteria
- Identify edge cases and error scenarios

### STEP 3: Research Design Trends and Best Practices
- Research current design trends for the application type
- Research award-winning designs in the domain
- Research design systems from leading companies (Material Design, Apple HIG, etc.)
- Research color psychology and accessibility
- Research typography best practices
- Research layout and spacing principles
- Research mobile-first and responsive design patterns

### STEP 4: Design Information Architecture
- Create site map or application structure
- Design navigation hierarchy
- Design information grouping and categorization
- Design search and filtering mechanisms
- Design content hierarchy and prioritization
- Design user flow diagrams for key scenarios

### STEP 5: Create Wireframes
- Create low-fidelity wireframes for all key screens
- Design layout structure and component placement
- Design navigation patterns
- Design input forms and data entry flows
- Design error states and empty states
- Design loading states and progress indicators
- Iterate on wireframes based on usability principles

### STEP 6: Design Visual System
- Design color palette with primary, secondary, accent, and semantic colors
- Design typography system (headings, body, captions, code)
- Design spacing and layout grid system
- Design component library (buttons, inputs, cards, modals, etc.)
- Design icon system and iconography
- Design elevation and shadow system
- Design border radius and corner radius system
- Document all design tokens

### STEP 7: Create Detailed UI Specifications
- Create high-fidelity specifications for all components
- Specify component states (default, hover, active, disabled, error, loading)
- Specify responsive behavior for each component
- Specify accessibility attributes (ARIA labels, focus states, keyboard navigation)
- Specify animation and transition specifications
- Specify component variants and use cases
- Create component documentation

### STEP 8: Design Responsive Layouts
- Design layouts for mobile (320px - 768px)
- Design layouts for tablet (768px - 1024px)
- Design layouts for desktop (1024px+)
- Design breakpoints and responsive strategies
- Design touch-friendly interactions for mobile
- Design keyboard-friendly interactions for desktop
- Ensure consistency across all screen sizes

### STEP 9: Design Accessibility Features
- Ensure all colors meet WCAG AA contrast requirements
- Design for keyboard navigation and focus indicators
- Design screen reader friendly interfaces
- Design for users with motor impairments
- Design for users with visual impairments
- Design for users with cognitive impairments
- Design skip links and landmark navigation
- Design accessible form labels and error messages

### STEP 10: Design Micro-Interactions and Animations
- Design button hover and click animations
- Design page transitions and route animations
- Design loading animations and skeleton screens
- Design success and error feedback animations
- Design scroll animations and parallax effects (if appropriate)
- Design gesture animations for mobile
- Ensure animations are performant and respect user preferences (prefers-reduced-motion)
