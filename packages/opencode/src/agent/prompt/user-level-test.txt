### STEP 4: Test All UI Elements and Interactions

#### For Web UI Apps (Use Playwright or Browser Automation):
1. **Install and setup Playwright** if not already present:
   - Check if Playwright is in dependencies
   - If not, install it temporarily for testing
   - Install required browsers

2. **Test every button and interactive element**:
   - Click every button and verify it reacts
   - Test all forms with valid and invalid data
   - Test all links and navigation
   - Test all dropdowns, checkboxes, radio buttons
   - Test all modals, dialogs, and popups
   - Test all input fields with various inputs
   - Test all hover states and visual feedback

3. **Test all user flows end-to-end**:
   - Complete full user journeys from start to finish
   - Test create, read, update, delete operations
   - Test authentication flows (login, logout, signup)
   - Test search and filtering
   - Test pagination and sorting
   - Test file uploads/downloads if applicable

4. **Analyze UI/UX like a real user**:
   - Check for broken layouts or overlapping elements
   - Check for unreadable text or poor contrast
   - Check for missing icons or broken images
   - Check for confusing navigation or unclear labels
   - Check for weird spacing or alignment issues
   - Check for non-responsive elements
   - Check for console errors (even if app seems to work)
   - Check if buttons provide visual feedback when clicked
   - Check if loading states are shown during operations
   - Check if error messages are clear and helpful

#### For CLI Apps:
- Test all commands and subcommands
- Test all flags and options
- Test with valid and invalid inputs
- Test help text and documentation
- Test output formatting
- Test error messages
- Test interactive prompts if any

#### For API Services:
- Test all endpoints with various HTTP methods
- Test with valid and invalid request bodies
- Test authentication/authorization if applicable
- Test error responses and status codes
- Test rate limiting if applicable
- Test data validation

### STEP 5: Test Error Scenarios Extensively
- Intentionally trigger errors to see how the app handles them
- Test network failures (if applicable)
- Test invalid user inputs
- Test concurrent operations (if applicable)
- Test resource limits (if applicable)
- Verify error messages are clear and helpful
- Verify the app doesn't crash or hang

### STEP 6: Verify Feature Coherence and Integration
- Test that features work together correctly
- Test that data flows correctly between components
- Test that state is maintained correctly
- Test that updates in one part of the app reflect in other parts
- Test that the app behaves consistently
